# Discord 익명 민원 관리 봇 사용방법

이 문서는 봇을 처음 설정하고 실행하는 순서를 정리한 문서입니다.  
기본 설정 파일은 `.env`입니다.

## 1. 준비물

- Node.js 20 이상
- Discord 봇 토큰
- 봇을 사용할 Discord 서버
- 민원 채널을 만들 카테고리
- 기록/알림을 받을 Discord 채널
- 관리자 역할 ID
- 최고관리자 사용자 ID

Discord ID를 복사하려면 Discord 설정에서 개발자 모드를 켠 뒤 서버, 채널, 역할, 사용자를 우클릭해서 ID를 복사하세요.

## 2. 파일 구성

```text
pansa/
  .env                  # 실제 실행 설정 파일
  .env.example          # 설정 예시 파일
  package.json          # 실행 스크립트와 의존성 정보
  src/                  # 봇 소스 코드
  keys/
    public.pem          # 기록 암호화용 공개키
  decrypt-gui/          # .enc 기록 복호화용 로컬 GUI
```

운영 중에는 아래 폴더가 자동으로 생길 수 있습니다.

```text
logs/                   # 봇 운영 로그
records/                # 종료된 민원 기록
sessions/               # Redis를 쓰지 않을 때의 세션 저장 파일
```

## 3. 설치

처음 한 번만 의존성을 설치합니다.

```bash
npm install
```

## 4. .env 설정

루트의 `.env` 파일을 열고 실제 값으로 바꿉니다.

필수로 바꿔야 하는 값:

```env
DISCORD_TOKEN=your-discord-bot-token
CLIENT_ID=123456789012345678
GUILD_ID=123456789012345678
COMPLAINT_CATEGORY_ID=123456789012345678
LOG_CHANNEL_ID=123456789012345678
ALERT_CHANNEL_ID=123456789012345678
ADMIN_ROLE_ID=123456789012345678
SUPER_ADMIN_ID=123456789012345678
SESSION_ENCRYPTION_KEY=replace-with-base64-encoded-32-byte-key
```

`SESSION_ENCRYPTION_KEY`는 아래 명령으로 새로 만들 수 있습니다.

```bash
node -e "console.log(require('node:crypto').randomBytes(32).toString('base64'))"
```

주의: 이 키를 바꾸면 기존 `sessions/*.enc.json` 파일을 읽을 수 없습니다.

## 5. 키 파일

봇 서버에는 `keys/public.pem`만 있어야 합니다.  
`private.pem`은 최고관리자 개인 PC나 별도 안전한 저장소에만 보관하세요.

키를 새로 만들고 싶으면 설정 마법사를 사용할 수 있습니다.

```bash
npm run setup
```

마법사는 `.env`, `keys/public.pem`, `private.pem`을 생성하거나 갱신합니다.  
생성된 `private.pem`은 봇 서버에서 빼서 별도로 보관하는 것을 권장합니다.

## 6. 설정 확인

실행 전에 환경 변수와 공개키 설정을 확인합니다.

```bash
npm run healthcheck
```

문법 체크만 하고 싶으면 아래 명령을 사용합니다.

```bash
npm run check
```

## 7. Discord 명령어 등록

봇의 슬래시 명령어를 서버에 등록합니다.

```bash
npm run deploy:commands
```

명령어가 Discord에 보이기까지 잠시 걸릴 수 있습니다.

## 8. 봇 실행

```bash
npm start
```

정상 실행되면 봇이 Discord에 로그인하고 민원 접수를 받을 준비가 됩니다.

## 9. 민원 처리 흐름

1. 사용자가 봇 DM에서 `/민원` 명령어를 실행합니다.
2. 개인정보 안내와 동의를 거칩니다.
3. 카테고리와 민원 내용을 입력합니다.
4. 서버의 지정 카테고리에 `complaint-익명ID` 채널이 생성됩니다.
5. 관리자는 생성된 채널에서 익명으로 사용자와 대화합니다.
6. 민원이 종료되면 기록 채널에 `.txt`와 `.enc` 기록이 올라갑니다.

`.txt` 기록에는 공개 가능한 대화 기록이 들어가고, `.enc` 기록에는 신원과 전체 기록 등 민감 정보가 암호화되어 들어갑니다.

## 10. 복호화 GUI 사용

복호화 GUI는 Node.js 없이 브라우저에서 열 수 있는 로컬 HTML 도구입니다.

사용 순서:

1. `decrypt-gui/index.html`을 브라우저로 엽니다.
2. Discord 기록 채널에서 받은 `complaint-XXXXXX.enc` 파일을 선택합니다.
3. 최고관리자 개인 PC에 보관한 `private.pem`을 선택합니다.
4. 복호화 사유를 입력합니다.
5. 복호화 버튼을 누르고 결과를 확인합니다.

감사 로그 웹훅을 쓰려면 `decrypt-gui/config.example.js`를 `decrypt-gui/config.js`로 복사한 뒤 Webhook URL을 넣습니다.

```js
window.DECRYPT_AUDIT_CONFIG = {
  webhookUrl: 'https://discord.com/api/webhooks/WEBHOOK_ID/WEBHOOK_TOKEN',
  operatorName: '최고관리자',
  requireAuditSuccess: true,
  privacyNotice: '복호화 결과에는 개인정보와 민감한 운영 정보가 포함될 수 있습니다.'
};
```

주의: `decrypt-gui/config.js`에는 실제 Webhook URL이 들어가므로 공유하거나 커밋하지 마세요.

## 11. 보안 주의사항

- `.env`에는 봇 토큰과 암호화 키가 들어가므로 외부에 공유하지 마세요.
- `private.pem`은 봇 서버에 두지 마세요.
- `records/*.enc`, `records/*.txt`, `sessions/*.enc.json`, `logs/*.log`는 운영 데이터이므로 공유 전에 반드시 확인하세요.
- 복호화한 `plain.json`에는 민원인 신원과 전체 대화가 포함될 수 있으므로 확인 후 안전하게 삭제하세요.
- `.env.example`에는 실제 값이 아닌 예시값만 넣으세요.

## 12. 자주 쓰는 명령어

```bash
npm install
npm run setup
npm run healthcheck
npm run deploy:commands
npm start
npm run check
```

CLI로 `.enc` 파일을 복호화해야 할 때:

```bash
node src/crypto/decrypt.js ./records/complaint-XXXXXX.enc ./private.pem ./plain.json
```

가능하면 CLI보다 `decrypt-gui/index.html`을 사용하고, 복호화 감사 로그를 남기는 방식을 권장합니다.
